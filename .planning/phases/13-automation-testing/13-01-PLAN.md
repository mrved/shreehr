2. Create `e2e/auth.spec.ts` with the following test structure using `test` and `expect` from `./fixtures/base`:

   ```
   describe('Authentication Flows')
     describe('Login')
       - 'should display login form with email and password fields'
         -> goto /login, assert emailInput, passwordInput, loginButton all visible
       - 'should login admin and redirect to /dashboard'
         -> use userRole: 'admin', login with admin creds, assert URL contains '/dashboard'
       - 'should login employee and redirect to employee portal'
         -> goto /login, login as employee, assert URL contains '/employee'
       - 'should login HR manager and redirect to /dashboard'
         -> use userRole: 'hr', login with hr creds, assert URL contains '/dashboard'
       - 'should show error for invalid credentials'
         -> goto /login, login with wrong password, assert error message visible, assert still on /login
       - 'should show error for non-existent email'
         -> goto /login, login with fake@test.com, assert error message visible

     describe('Logout')
       - 'should logout admin and redirect to login page'
         -> use authenticatedPage as admin, find and click logout, assert redirected to /login
       - 'should not access dashboard after logout'
         -> login as admin, logout, try navigating to /dashboard, assert redirected to /login

     describe('Role-Based Access')
       - 'should redirect employee accessing /dashboard to employee portal'
         -> login as employee, navigate to /dashboard, assert redirected to /employee or access denied
       - 'should allow admin to access /dashboard/employees'
         -> login as admin, navigate to /dashboard/employees, assert page loads (heading visible)
       - 'should allow HR to access /dashboard/employees'
         -> login as hr, navigate to /dashboard/employees, assert page loads
   ```

   For each test: use explicit assertions (toHaveURL, toBeVisible), respect Playwright best practices (auto-waiting built in), use page object methods where possible.
