Create `e2e/payroll.spec.ts` using `test` and `expect` from `./fixtures/base`.
All tests use `userRole: 'admin'`.

```
describe('Payroll Processing')
  test.describe.configure({ mode: 'serial' })

  describe('Payroll Dashboard')
    - 'should display payroll dashboard'
      -> navigate to /dashboard/payroll, assert page heading visible, assert run payroll button visible

  describe('Salary Structures')
    - 'should display salary structures page'
      -> navigate to /dashboard/payroll/salary-structures, assert list visible, assert seeded employees have salary structures

  describe('Run Payroll')
    NOTE: This is the CRITICAL test. Payroll runs are async (BullMQ background jobs).
    This test requires Redis to be running. If Redis is not available, tests should skip gracefully.

    - 'should navigate to run payroll page'
      -> click run payroll button, assert URL contains '/run', assert month/year selectors visible
    - 'should validate payroll prerequisites'
      -> select a past month (e.g., 2 months ago), click validate, assert validation results shown
    - 'should initiate payroll run'
      -> after validation passes, click run button, assert processing starts (progress/status visible)
      -> NOTE: If validation fails (e.g., attendance not locked), this is expected behavior. Assert the validation message is informative.
    - 'should show payroll run completion'
      -> wait for processing to complete (up to 60s), assert status shows COMPLETED or verify run appears in recent runs list

  describe('Payroll Records and Payslips')
    - 'should display payroll records for completed run'
      -> navigate to a completed payroll run detail page, assert records table visible, assert at least one record
    - 'should show employee payroll details'
      -> in records table, verify employee name, gross, deductions, net pay columns are visible
    - 'should download payslip PDF'
      -> click download payslip for an employee, assert download starts (use page.waitForEvent('download'))

  describe('Statutory Files')
    - 'should generate ECR file'
      -> on run detail page, click generate ECR, assert file download starts or success message shown
    - 'should generate ESI challan'
      -> on run detail page, click generate ESI challan, assert download starts or success message shown
```

Implementation notes:
- Payroll processing is async (BullMQ). Use extended timeout (60s+) for waitForProcessingComplete.
- If no completed payroll run exists (fresh database), the records/payslip tests may need to create one first. Combine run + records in a serial block.
- For download assertions: use `const download = await page.waitForEvent('download')` then `expect(download.suggestedFilename()).toContain('.pdf')` or `.toContain('.txt')`
- If Redis is not running, payroll will fail. Add a check: if validate returns Redis error, mark test as skipped with `test.skip()`.
- Past month selection avoids interference with current month attendance.
