```
describe('Leave Management')
  describe('Leave Balances')
    test.use({ userRole: 'employee' })

    - 'should display leave balance page for employee'
      -> navigate to /employee/leave, assert leave balance section visible
    - 'should show balances for configured leave types'
      -> navigate to /employee/leave, assert at least one leave type balance card visible (Casual Leave, Sick Leave, or Earned Leave from seed data)

  describe('Apply for Leave')
    test.use({ userRole: 'employee' })

    - 'should navigate to leave application form'
      -> from /employee/leave, click apply button, assert form visible
    - 'should apply for casual leave successfully'
      -> fill form: type=Casual Leave, startDate=future date (2 weeks from now, Monday), endDate=same day, reason="E2E test leave". Submit. Assert success message or redirect.
    - 'should show validation error for past dates'
      -> fill form with past start date, submit, assert error message visible
    - 'should calculate leave days correctly'
      -> fill start and end dates spanning 3 weekdays, assert leave days count shows 3 (excluding weekends)

  describe('Leave Approval Workflow')
    test.describe.configure({ mode: 'serial' })

    NOTE: This is the critical multi-role workflow test.

    Step 1 (employee applies):
    - 'employee should apply for leave'
      -> test.use({ userRole: 'employee' }), apply for casual leave for a future date, note the date used

    Step 2 (admin approves):
    - 'admin should see pending leave request'
      -> test.use({ userRole: 'admin' }), navigate to /dashboard/leave, assert pending request from "Test Employee" visible
    - 'admin should approve leave request'
      -> find Test Employee's request, click approve, assert status changes to "Approved"

    Step 3 (employee verifies):
    - 'employee should see approved leave in their requests'
      -> test.use({ userRole: 'employee' }), navigate to /employee/leave, assert request shows "Approved" status

  describe('Leave Rejection')
    test.use({ userRole: 'admin' })

    - 'admin should reject leave request with reason'
      -> (if a pending request exists) click reject on a request, fill rejection reason, confirm. Assert status changes to "Rejected".

  describe('Admin Leave Types')
    test.use({ userRole: 'admin' })

    - 'should display configured leave types'
      -> navigate to /dashboard/leave/types, assert leave types list visible, assert at least 3 types (from seed data)
```

Implementation notes:
- For the multi-role approval workflow: since Playwright fixtures create separate authenticated sessions, the serial describe block needs to manage auth explicitly. Use `test.use({ userRole })` per test OR login/logout within tests.
- Future dates: compute dates dynamically. Use a Monday 2-3 weeks from now to avoid weekend issues.
- Balance validation after approval is ideal but may be complex (race conditions). Assert status change is sufficient.
- Use `page.waitForResponse('**/api/leave-requests**')` after form submissions
