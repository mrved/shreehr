  <name>Task 1: Create Announcement API routes (CRUD + org-wide email)</name>
  <files>src/app/api/announcements/route.ts, src/app/api/announcements/[id]/route.ts</files>
  <action>
**Create `src/app/api/announcements/route.ts`:**

**GET handler** — List active (non-archived) announcements.
- Auth required (any authenticated user)
- Query `prisma.announcement.findMany` where `is_archived: false`, ordered by `created_at: 'desc'`, take 10
- Select: id, title, content, is_archived, created_at, author (select: name)
- Return `{ announcements }` with 200

**POST handler** — Create announcement + queue org-wide email.
- Auth required. RBAC: Only `ADMIN`, `SUPER_ADMIN`, `HR_MANAGER` can create.
- Validate body with Zod: `{ title: z.string().min(1).max(200), content: z.string().min(1).max(5000) }`
- Create announcement with `prisma.announcement.create({ data: { title, content, created_by: session.user.id } })`
- Call `invalidateAnnouncements()` from cache.ts
- Query all active employees with user email: `prisma.employee.findMany({ where: { employment_status: 'ACTIVE' }, include: { user: { select: { email: true } } } })`
- For each employee with a user email, call `addEmailJob('announcement-notification', email, { employeeName: first_name, title, content, postedBy: session.user.name, dashboardUrl: process.env.NEXT_PUBLIC_APP_URL || 'http://localhost:3000' })`
- Use `Promise.all` for the email jobs (they're just queue adds, very fast)
- Return `{ announcement, emailsQueued: count }` with 201

**Create `src/app/api/announcements/[id]/route.ts`:**

**PATCH handler** — Archive/unarchive announcement.
- Auth required. RBAC: Only `ADMIN`, `SUPER_ADMIN`, `HR_MANAGER`.
- Validate body with Zod: `{ is_archived: z.boolean() }`
- Update announcement with `prisma.announcement.update`
- Call `invalidateAnnouncements()`
- Return updated announcement with 200

**DELETE handler** — Delete announcement permanently.
- Auth required. RBAC: Only `ADMIN`, `SUPER_ADMIN`.
- Delete with `prisma.announcement.delete`
- Call `invalidateAnnouncements()`
- Return 204

IMPORTANT: Never return employee PII (encrypted fields) in any response. Only return author name, not author email or other details.

  <name>Task 2: Create Poll API routes (CRUD + voting)</name>
  <files>src/app/api/polls/route.ts, src/app/api/polls/[id]/route.ts, src/app/api/polls/[id]/vote/route.ts</files>
  <action>
**Create `src/app/api/polls/route.ts`:**

**GET handler** — List active (non-closed) polls with results.
- Auth required (any authenticated user)
- Query `prisma.poll.findMany` where `is_closed: false`, ordered by `created_at: 'desc'`
- Include: `options: { include: { _count: { select: { responses: true } } }, orderBy: { order: 'asc' } }`, `_count: { select: { responses: true } }`, `author: { select: { name: true } }`
- Also include the current user's vote if they have one: For each poll, check if the current employee has a PollResponse. Use a separate query: `prisma.pollResponse.findMany({ where: { employee_id: session.user.employeeId, poll_id: { in: pollIds } } })`
- Merge the user's vote into the response as `myVote: { optionId: string } | null` per poll
- Return `{ polls }` with 200

**POST handler** — Create poll with options.
- Auth required. RBAC: Only `ADMIN`, `SUPER_ADMIN`, `HR_MANAGER`.
- Validate body with Zod: `{ title: z.string().min(1).max(200), description: z.string().max(1000).optional(), options: z.array(z.string().min(1).max(200)).min(2).max(10), closes_at: z.string().datetime().optional() }`
- Create poll with nested options: `prisma.poll.create({ data: { title, description, closes_at, created_by: session.user.id, options: { create: options.map((label, i) => ({ label, order: i })) } }, include: { options: true } })`
- Call `invalidatePolls()` from cache.ts
- Return `{ poll }` with 201

**Create `src/app/api/polls/[id]/route.ts`:**

**GET handler** — Poll detail with full results.
- Auth required
- Fetch poll with options, response counts, and total responses
- Include author name
- If the current user is an employee, include their vote
- Return `{ poll }` with 200

**PATCH handler** — Close poll.
- Auth required. RBAC: Only `ADMIN`, `SUPER_ADMIN`, `HR_MANAGER`.
- Validate body: `{ is_closed: z.boolean() }`
- Update poll
- Call `invalidatePolls()`
- Return updated poll with 200

**Create `src/app/api/polls/[id]/vote/route.ts`:**

**POST handler** — Cast or change vote.
- Auth required. Must have employeeId (employee role).
- Validate body: `{ optionId: z.string() }`
- Verify the poll exists and is not closed. If `poll.closes_at` is set and past, return 400 "Poll has closed".
- Verify the optionId belongs to this poll.
- Use `prisma.pollResponse.upsert` with:
  - `where: { poll_id_employee_id: { poll_id: params.id, employee_id: session.user.employeeId } }`
  - `update: { option_id: optionId }`
  - `create: { poll_id: params.id, employee_id: session.user.employeeId, option_id: optionId }`
- Call `invalidatePolls()`
- Return `{ success: true, vote }` with 200

IMPORTANT: The upsert pattern ensures one vote per employee per poll. Changing votes is allowed (upsert replaces the option_id).
