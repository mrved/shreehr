**Create `src/lib/dashboard/birthdays.ts`:**

Pure functions that operate on employee data (no Prisma dependency — receive employee arrays for testability).

```typescript
import { addDays } from 'date-fns';

interface EmployeeDateInfo {
  id: string;
  first_name: string;
  last_name: string;
  date_of_birth: Date;
  date_of_joining: Date;
}

interface UpcomingBirthday {
  id: string;
  name: string;
  date: Date;       // This year's birthday date
  daysUntil: number;
}

interface UpcomingAnniversary {
  id: string;
  name: string;
  date: Date;        // This year's anniversary date
  daysUntil: number;
  yearsOfService: number;
}

interface TodayCelebration {
  name: string;
  type: 'birthday' | 'anniversary';
  years?: number;
}
```

**`getUpcomingBirthdays(employees, windowDays = 30)`:**
- For each employee, compute this year's birthday: `new Date(today.getFullYear(), bday.getMonth(), bday.getDate())`
- If this year's birthday has already passed, use next year
- Filter: birthday is between today and today + windowDays (inclusive)
- Sort by daysUntil ascending
- Return `UpcomingBirthday[]`

**Handle the December-January boundary:** When today is Dec 20 and window is 30 days, `windowEnd = Jan 19 (next year)`. A birthday on Jan 5 should be included because after `setFullYear(currentYear)`, Jan 5 < Dec 20, so we set to next year, making it Jan 5 next year which is < Jan 19 next year.

**`getUpcomingAnniversaries(employees, windowDays = 30)`:**
- Same window logic as birthdays but using `date_of_joining`
- Exclude employees whose `date_of_joining` year equals current year (not yet 1 year)
- Calculate `yearsOfService` from difference between current year and joining year
- Sort by daysUntil ascending

**`getTodayCelebrations(employees)`:**
- Call both getUpcomingBirthdays and getUpcomingAnniversaries with windowDays=0
- Actually simpler: filter for daysUntil === 0 from both
- Return `TodayCelebration[]`
